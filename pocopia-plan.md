# 포코피아 (Pocopia) - 프로젝트 계획서

> 닌텐도 스위치2 게임 「포코피아」의 써드파티 클라우드섬 공유 사이트

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | 포코피아 (Pocopia) |
| **설명** | 사용자가 본인의 클라우드섬 정보를 업로드하면, 다른 사용자가 코드를 보고 접속할 수 있는 간단한 공유 플랫폼 |
| **타겟 게임** | 닌텐도 스위치2 - 포코피아 |
| **호스팅** | 개인 맥북 (로컬 서버) |
| **예산** | 물리 서버 및 클라우드 인프라 비용 0원 (묣) |

---

## 2. 핵심 요구사항

### 2.1 기능 요구사항

1. **큐우드섬 업로드**
   - 사용자는 다음 정보를 제출할 수 있어야 함:
     - 제목 (간단한 이름)
     - 설명 (선택적, 짧은 소개)
     - 8자리 클라우드섬 코드 (숫자/알파벳 조합)
   - 업로드 시 해당 게시물은 **60초 동안만 유효**해야 함
   - 60초 경과 후 게시물은 **자동으로 삭제**되어야 함

2. **큐우드섬 탐색 및 접속**
   - 모든 활성화된(60초 이내 업로드된) 클라우드섬 목록을 실시간으로 확인 가능해야 함
   - 각 게시물에서 8자리 코드를 확인할 수 있어야 함
   - 사용자는 해당 코드를 보고 직접 스위치2에서 클라우드섬에 접속

3. **실시간성**
   - 새로운 게시물 업로드 시 목록에 즉시 반영되어야 함
   - 60초 만료된 게시물은 목록에서 즉시 제거되어야 함

### 2.2 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| **비용** | 모든 서비스 및 도구 묣 사용 |
| **서버 환경** | 맥북 로컬 (macOS) |
| **동시 접속자** | 소규모 (10~50명 내외 가정) |
| **데이터 영속성** | 불필요 - 60초 후 자동 삭제 |
| **인증/회원가입** | 불필요 - 익명 업로드/엽람 |
| **API 연동** | 닌텐도/스위치 API 없음 (코드 수동 입력) |

## 3. 기술 스택

### 3.1 최종 선정 스택 (추천)

| 구분 | 기술 | 선정 이유 |
|------|------|-----------|
| **백엔드** | Python + Flask | 설치 간편, 코드 간결, 문서 풍부 |
| **데이터 저장** | 메모리 내장 구조 (Python dict) | 60초 TTL 관리에 최적, DB 불필요 |
| **프론트엔드** | 순수 HTML + CSS + JavaScript | 빌드 도구 불필요, 가장 가벼움 |
| **외부 접근** | ngrok (묣 플랜) | HTTPS 터널 즉시 발급, 설정 1분 |
| **TTL 관리** | Python threading.Timer | 간단한 타이머 기반 자동 삭제 |

### 3.2 대안 스택

| 구분 | 대안 | 사용 시기 |
|------|------|-----------|
| 백엔드 | Node.js + Express | JavaScript/Node 생태계가 더 익숙할 때 |
| 외부 접근 | Cloudflare Tunnel | ngrok 묣 플랜 제한(월 1GB) 초과 시 |
| 프론트엔드 | React/Vue | 확장성이 필요해질 때 (현재 과함) |

## 5. API 설계

### 5.1 엔드포인트

| 메서드 | 경로 | 설명 | 요청 본문 | 응답 |
|--------|------|------|-----------|------|
| `GET` | `/islands` | 활성화된 클라우드섬 목록 조회 | - | `[{id, title, description, code, created_at, expires_at}]` |
| `POST` | `/islands` | 새 클라우드섬 업로드 | `{"title": "...", "description": "...", "code": "1234ABCD"}` | `{id, title, description, code, created_at, expires_at}` |
| `DELETE` | `/islands/<id>` | 특정 클라우드섬 수동 삭제 (선택적) | - | `{"success": true}` |

### 5.2 데이터 모델

```python
island = {
    "id": "uuid-string",           # 고유 식별자
    "title": "string",              # 제목 (최대 50자)
    "description": "string",        # 설명 (최대 200자, 선택)
    "code": "string",               # 8자리 클라우드섬 코드
    "created_at": "ISO-8601",       # 생성 시각
    "expires_at": "ISO-8601"        # 만료 시각 (created_at + 60초)
}
```

### 5.3 유효성 검증

- `title`: 필수, 1~50자
- `description`: 선택, 0~200자
- `code`: 필수, 정확히 8자리 (알파벳 대문자 + 숫자)

### 6.2 동작 흐름

1. 페이지 로드 시 `/islands` API 호출 → 활성 목록 표시
2. 사용자가 폼 작성 후 제출 → `POST /islands` → 성공 시 목록 갱신
3. 1초마다 남은 시간 카운트다운 업데이트
4. 남은 시간 0초 도달 시 해당 카드 DOM에서 제거
5. (선택) WebSocket 또는 폴링으로 실시간 목록 동기화

## 7. 구현 계획

### 7.1 파일 구조

```
pocopia/
├── app.py              # Flask 서버 (API + 정적 파일 서빙)
├── requirements.txt    # Python 의존성
├── static/
│   ├── index.html      # 메인 페이지
│   ├── style.css       # 스타일시트
│   └── app.js          # 프론트엔드 로직
└── README.md           # 실행 방법
```

### 7.2 구현 단계

| 단계 | 작업 내용 | 예상 소요시간 |
|------|-----------|---------------|
| 1 | 프로젝트 폴더 생성 및 Flask 설치 | 5분 |
| 2 | `app.py` - 기본 서버 및 API 구현 | 20분 |
| 3 | `index.html` - 페이지 구조 작성 | 15분 |
| 4 | `style.css` - 기본 스타일링 | 15분 |
| 5 | `app.js` - API 연동 및 동적 UI | 20분 |
| 6 | ngrok 설치 및 터널링 테스트 | 10분 |
| 7 | 전체 통합 테스트 | 15분 |
| **총계** | | **약 100분 (1시간 40분)** |

### 7.3 핵심 구현 포인트

#### 백엔드 (app.py)
- 전역 변수 `islands = {}` 로 메모리 저장소 구현
- `POST /islands` 시:
  1. UUID 생성
  2. 현재 시각 + 60초 = `expires_at` 설정
  3. `islands[id] = data` 저장
  4. `threading.Timer(60, lambda: islands.pop(id, None)).start()`
- `GET /islands` 시:
  1. 현재 시각 기준 `expires_at` 지난 항목 필터링
  2. 정렬 (최신순) 후 반환

#### 프론트엔드 (app.js)
- 폼 제출 시 `fetch()`로 POST 요청
- 주기적 폴링 (`setInterval(fetchIslands, 5000)`) 또는 WebSocket
- 카운트다운 타이머: `setInterval`로 1초마다 `expires_at - now` 계산

## 8. 실행 방법

### 8.1 로컬 실행

```bash
# 1. 프로젝트 폴더로 이동
cd pocopia

# 2. 가상환경 생성 (선택)
python -m venv venv
source venv/bin/activate

# 3. 의존성 설치
pip install -r requirements.txt

# 4. 서버 실행
python app.py
# → http://localhost:5000 에서 접속 가능
```

### 8.2 외부 접근 (ngrok)

```bash
# 1. ngrok 설치 (Homebrew)
brew install ngrok

# 2. ngrok 계정 가입 후 authtoken 설정
ngrok config add-authtoken <YOUR_TOKEN>

# 3. 터널 실행
ngrok http 5000

# 4. 발급된 HTTPS URL을 사용자들에게 공유
# 예: https://abcd1234.ngrok.io
```

## 9. 제약사항 및 고려사항

| 항목 | 내용 |
|------|------|
| **ngrok 묣 플랜 제한** | 월 1GB 데이터 전송, 세션당 2시간 제한 |
| **서버 가동 시간** | 맥북이 켜져 있어야 접속 가능 |
| **데이터 유실** | 서버 재시작 시 모든 게시물 초기화 (의도된 동작) |
| **동시 접속 한도** | Flask 개발 서버 기준 수십~수백 동시 접속 가능 |
| **보안** | 익명 업로드로 인한 스팸 가능성 → 필요시 Rate Limiting 추가 |
